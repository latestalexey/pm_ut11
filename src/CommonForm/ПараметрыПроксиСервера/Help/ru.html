Для доступа в Интернет с клиентских компьютеров параметры прокси‑сервера задаются на каждом клиентском рабочем месте (по команде Персональные настройки в разделе Настройка и администрирование).

При использовании клиент‑серверной версии 1С:Предприятия параметры прокси‑сервера также указываются и для сервера 1С:Предприятия (форма настроек параметров системы в разделе Настройка и администрирование). В общем случае, параметры прокси-сервера для клиентских рабочих мест и сервера 1С:Предприятия могут различаться.

Особенности настройки параметров прокси-сервера на сервере 1С:Предприятия
Параметры прокси-сервера для сервера 1С:Предприятия хранятся на серверном компьютере в файле inetcfg.xml, расположенном в каталоге конфигурационных файлов версии платформы 1С:Предприятие. Каталог конфигурационных файлов версии платформы 1С:Предприятия располагается по адресу "<Каталог платформы>\conf\", например "C:\Program Files\1cv82\8.2.16.362\bin\conf\". Если в файле "<Каталог платформы>\conf\conf.cfg" задан параметр "ConfLocation", то версия платформы использует общий каталог конфигурационных файлов для всех версий платформы, установленных на компьютере (как правило, это "C:\Program Files\1cv82\conf\" или "C:\Program Files\1cv8\conf\").

При настройке параметров прокси-сервера на сервере 1С:Предприятия, нужно иметь в виду, что настройки прокси-сервера будут действовать:
- только для информационных баз под управлением текущей версии платформы 1С:Предприятие, если версия платформы использует собственный каталог конфигурационных файлов;
- для всех информационных баз под управлением любой версии платформы 1С:Предприятие, установленной на сервере, если на серверном компьютере используется общий каталог конфигурационных файлов для всех установленных версий платформы 1С:Предприятие.

В файловом варианте серверные параметры прокси-сервера хранятся в каталоге конфигурационных файлов платформы на компьютере пользователя и действуют:
- для всех файловых информационных баз, работа с которыми осуществляется на текущей версии платформы 1С:Предприятия, если версия платформы использует собственный каталог конфигурационных файлов;
- для всех файловых информационных баз, работа с которыми осуществляется на любой версии платформы 1С:Предприятия, если используется общий каталог конфигурационных файлов для всех установленных версий платформы 1С:Предприятие.

Если у платформы 1С:Предприятие (на серверном или на пользовательском компьютере в файловом или клиент-серверном режиме работы соответственно) отсутствуют права на изменение файла inetcfg.xml, то параметры прокси-сервера для сервера 1С:Предприятия будут действовать только для тех функций программы, которые обращаются к сети Интернет без использования веб-сервисов. Данная ситуация не критична и требует исправления в том случае, если необходимо настроить параметры прокси-сервера еще и для тех функций программы, которые рассчитанны на работу с веб-сервисами. Для этого в зависимости от режима работы информационной базы (клиент-серверный или файловый) на серверном или пользовательском компьютере 1С:Предприятия требуется предоставить права на запись файла inetcfg.xml пользователю операционной системы, от имени которого работает платформа 1С:Предприятие.
