Документ "Заказ клиента"

Заказ клиента - это запрос клиента на поставку ему товаров или оказание услуг в установленные сроки. Заказ клиента может быть оформлен на основании данных о первичном спросе клиента, зарегистрированном в документе "Сделка" или на основании согласованного с клиентом коммерческого предложения.

В заказе клиента обязательно должно быть указано соглашение. В соответствии с этим соглашением (типовым или индивидуальным) регистрируются те условия, по которым будет осуществляться отгрузка и оплата товара по заказу. На основании данных, зарегистрированном в соглашении (цены, графики оплат, условия поставки) и даты, указанной в заказе, рассчитывается и заполняется информация об условиях продажи товаров по заказу: указывается вид цен на номенклатурные позиции, желаемая дата оплаты, этапы оплаты с указанием дат и сумм оплат по каждому этапу.

В процессе работы менеджер может указать различные статусы оформления заказа: не согласован, согласован, к обеспечению, к отгрузке, закрыт. При переходе заказа из одного статуса в другой контролируется состояние заказа. Информация о текущем состоянии заказа также доступна менеджеру. Например, заказ не может быть принят к обеспечению, если не выполнены условия по его оплате до момента обеспечения (например, не оформлен авансовый платеж). При работе с заказами можно установить приоритетность выполнения заказов. В дальнейшем при работе с заказами пользователь может сортировать заказы в зависимости от их приоритета. В процессе работы по заказам менеджер может отменить отдельные позиции по заказу, После окончания работы по заказу пользователь должен установить для заказа статус "Закрыт". Если заказ выполнен не полностью, то пользователь должен указать причину, по которой невозможно было обеспечить заказ полностью (причина отмены).

  • При регистрации заказа со статусом "Согласован" проверяется соответствие условий, установленных в заказе, тем условиям, которые зарегистрированы в соглашении. В том случае, если эти условия отличаются, то данный заказ должен быть отправлен на дополнительное согласование.
  • Распоряжением службе логистики на обеспечение товаров является проведение заказа с установленным статусом "К обеспечению". Программа позволяет менеджеру автоматически сформировать список заказов клиентов, по которым можно принимать решение о выдаче распоряжений на обеспечение товаров, то есть отобрать те заказы, по которым выполнены все условия по предварительной оплате товаров в соответствии с условиями, указанными в соглашении. При выдаче распоряжений на обеспечение программа помогает пользователю рассчитать предполагаемую дату отгрузки по товарам, указанным в заказе. Расчет предполагаемой даты отгрузки зависит от того варианта контроля остатков, который установлен для склада, с которого предполагается отгрузка. Расчет может производиться в соответствии с текущими остатками на складе, с учетом уже зарезервированных товаров, а также с учетом планируемых поступлений товаров. После проведения заказа со статусом "К обеспечению" информация о данном заказе попадает в службу логистики. Служба логистики обеспечивает поставку товаров к указанной дате.
  • Распоряжением на отгрузку товаров является проведение заказа клиента со статусом "К отгрузке". При проведении документа контролируется выполнение условий по оплате, которые должны быть выполнены до момента отгрузки товаров по заказу. В процессе работы с заказом менеджер может контролировать правильность выполнения службой логистики распоряжений на отгрузку товаров: правильность оформления финансовых документов и правильность фактической отгрузки товаров по заказу.
