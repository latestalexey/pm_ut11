Как создать заказ на внутреннюю передачу товаров между обособленными подразделениями?

Заказ на внутреннюю передачу товаров между обособленными подразделениями оформляется аналогично заказу на перемещение товаров. В документе на странице "Дополнительно" указывается вид операции "Внутреннняя передача товаров", а также дополнительно указываются те обособленные подразделения (Организация-получатель и Организация отправитель), между которыми надо оформить перемещение товаров.

Важно. Оформить данную операцию можно только в том случае, если в программе ведется учет в разрезе обособленных подразделений, выделенных на отдельный баланс (установлена функциональная опция "Обособленные подразделения на отдельном балансе" в разделе "Администрирование/Финансы").

Работа с документом "Заказ на перемещение"

Заполнение документа "Заказа на перемещение"

Какая организация должна быть указана в заказе на перемещение?

В новом документе выбирается та организация, от имени которой оформляется "Заказ на перемещение". В заказе на перемещение может быть выбрана любая организация, которая указана в справочнике "Организации".

Как указать склад-отправитель, с которого перемещается товар?

Информация о складе, с которого необходимо переместить товар заполняется в поле "Отправитель". Из списка складов выбирается тот склад, с которого необходимо переместить товар.

Как указать склад, на который должен поступить товар?

Эта информация заполняется в поле "Получатель". Из списка складов выбирается тот склад, на который необходимо переместить товар.

Какие статусы устанавливаются в заказе на перемещение?

В заказе на перемещение можно указать три статуса: "К обеспечению", "К выполнению" и "Закрыт".

Если по заказу существует потребность и ее необходимо обеспечить к определенной дате, то устанавливается статус "К обеспечению". При проведении заказа в статусе "К обеспечению" контролируется наличие товаров на складе-отправителе, указанном в документе. В документе по каждому товару необходимо указать следующие даты.

  • Дата, когда товар должен поступить на склад-получатель ("Окончание поступления").
  • Дата, когда товар должен быть отгружен со склада отправителя ("Начало отгрузки").

Если потребность по заказу уже обеспечена: товар есть в наличии на складе-отправителе и его можно переместить на склад-получатель к указанной дате поступления, то устанавливается статус "К выполнению".

Важно. В программе предусмотрена возможность автоматического заполнения дат начала отгрузки и окончания поступления товаров. Даты могут быть заполнены автоматически в соответствии с произвольной датой, указанной пользователем или рассчитаны автоматически на основании желаемой даты поставки на склад-получатель и дней, которые необходимы для перемещения товаров со склада на склад.

Как указать в заказе на перемещение информацию о той дате, когда товар должен поступить на склад-получатель?
  • В поле "Желаемая дата поступления" укажите ту дату, когда товар должен поступить на склад-получатель.
  • В поле "Длительность перемещения" укажите то количество дней, которое необходимо для перемещения товаров со склада на склад.

На основании этих данных будут рассчитаны даты поступления и даты отгрузки для каждого товара.

Как заполнить список товаров, которые необходимо переместить с одного склада на другой?

  • В табличной части документа нажмите на кнопку "Добавить".
  • Список товаров, которые необходимо переместить, добавляется путем выбора из справочника "Номенклатура".
  • Если для товара ведется учет в разрезе характеристик и упаковок товара, до добавьте для товара информацию о его характеристике или упаковке.
  • Укажите количество перемещаемого товара.
  • Для каждого товара укажите даты начала отгрузки со склада-отправителя и даты окончания поступления товаров на склад-получатель. Информация об этих датах может быть заполнена автоматически.

Как рассчитываются даты начала отгрузки и даты окончания поступления для каждого товара?

Для расчета даты начала отгрузки и даты окончания поступления для каждого товара выполните следующие действия.

  • В командной панели табличной части документа нажмите на кнопку "Функции".
  • Выберите пункт "Рассчитать дату окончания поступления".

Даты будут рассчитаны по следующему алгоритму.

  • На основании указанной желаемой даты поступления и длительности перемещения будет рассчитана желаемая дата отгрузки.
  • Для каждого товара производится сравнение рассчитанной желаемой даты отгрузки с той датой, когда товар может быть отгружен со склада-отправителя в соответствии с графиком движения товаров.
  • Если товар нельзя отгрузить к рассчитанной желаемой дате отгрузки, то эта дата корректируется и в соответстви с ней уточняется дата окончания поступления товаров на склад.

Можно ли для нескольких товаров указать произвольные даты отгрузки и поступления?
  • В табличной части документа отметьте те товары, по которым необходимо изменить даты окончания поступления и даты начала отгрузки.
  • Нажмите на кнопку "Функции".
  • Выберите пункт "Заполнить дату окончания поступления для выделенных строк".
  • В появившемся диалоговом окне укажите дату, когда отмеченные товары должны поступить на склад-получатель.
  • Нажмите на кнопку "ОК".
  • Для отмеченных товаров заполнится дата окончания поступления товаров. Дата начала отгрузки при этом будет автоматически рассчитана с учетом количества дней, необходимых для перемещения товаров ("Длительность перемещения").
  • Аналогичным образом заполните даты для других товаров.

Важно. Если для всех перемещаемых товаров необходимо указать одну и ту же дату окончания поступления товаров и дату окончания отгрузки, то можно использовать команду "Заполнить дату окончания поступления для всех строк".

Регистрация заказа на перемещение

Регистрация заказа производится при выполнении команды "Провести". Перед проведением заказа контролируется заполнение всех обязательных реквизитов: организация, склад-отправитель, склад-получатель, дата отгрузки, дата поступления ответственный и т.д. При проведении документа с установленным статусом "К выполнению" контролируется наличие перемещаемых товаров в свободном остатке.

Как в документе можно получить данные из терминала сбора данных?

Для того чтобы загрузить данные из терминала сбора данных в табличную часть "Товары" необходимо воспользоваться командой табличной части "Все действия" - "Загрузить товары из ТСД". Если терминал сбора данных подключен и настроены параметры его подключения, то табличная часть заполниться данными из терминала. В противном случае программа выдаст предупреждение. Если из теминала будет получен штрих код не зарегистрированый в информационной базе, то программа предложит зарегистрировать его и сопоставить с имеющимся в информационной базе товаром.

Как можно добавить данные в табличную часть "Товары" с помощью сканера?

Если сканер подключен и настроены параметры его подключения, то после считывания штрих кода в табличную часть будет добавлен товар соответствующий штрих коду. Если товар уже присутствует в табличной части, то будет увеличено его количество. Если в программе не зарегистрирован считываемый штрих код, то программа предложит зарегистрировать его и сопоставить имеющемуся в информационной базе товару.

Как найти в табличной части товар используя сканер штрих кода?

Для того что бы выполнить поиск товара в табличной части, необходимо воспользоваться командой табличной части "Все действия" - "Найти товары по штрих коду". Далее в открывшейся форме считать штрих код товара. Если товар со считанным штрих кодом присутствует в табличной части, то он будет выделен (на него будет установлен курсор). Если в программе не зарегистрирован считываемый штрих код, то программа предложит зарегистрировать его и сопоставить имеющемуся в информационной базе товару.
