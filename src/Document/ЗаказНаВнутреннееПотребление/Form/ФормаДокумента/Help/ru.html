Заполнение заказа на внутреннее потребление

Какие виды операций предусмотрены в заказах на внутреннее потребление?

В заказе на внутреннее потребление можно выбрать следующие виды операций.

  • Списание на расходы - операция выбирается в том случае, если переданные для внутреннего использования товары планируется списать на определенную статью расходов.
  • Передача в эксплуатацию - операция выбирается в том случае, если товары передаются во временное использование конкретным сотрудникам и необходимо контролировать факт передачи (например, передача спецодежды).
  • Передача на прочие цели - операция оформляется в том случае, если факт передачи для внутреннего использования не должен отражаться в финансовом результате.

Как выбрать организацию от имени которой оформляется заказ на внутреннее потребление?

Информация об организации, по которой оформляется заказ, указывается в поле "Организация". Пользователь может выбрать любую организацию. Для быстрого выбора организации в поле выбора можно указать ИНН организации.

Как указать склад, с которого будет производиться отгрузка товаров для внутреннего потребления?

В поле "Склад" указывается тот склад, на котором должно быть обеспечено наличие товара для последующей отгрузки товаров со склада.

Кто указывается в качестве заказчика товаров в документе "Заказ на внутреннее потребление"?

В качестве заказчика для внутреннего потребления товаров указывается подразделение, у которого возникла потребность в получении товаров на внутренние нужды.

Как добавить новую номенклатурную позицию?

Новая номенклатурная позиция добавляется путем выбора из справочника "Номенклатура" (кнопка "Добавить"). Для добавления нескольких позиций в табличную часть документа можно также использовать диалоговое окно подбора (кнопка "Подобрать товары").

  • Если для товара, указанного в заказе, ведется учет в разрезе характеристик, то дополнительно вводится информация о характеристике товара.
  • Для каждого товара указывается количество, которое заказывается. Из выпадающего списка выбирается упаковка, в которой заказывается товар. Если товар заказывается без упаковки, то из списка выбирается <без упаковки>.

Какие статусы могут быть указаны в заказе на внутреннее потребление?

Если по заказу существует потребность и ее необходимо обеспечить к определенной дате, то устанавливается статус "К обеспечению" и рассчитывается та дата отгрузки, к которой необходимо обеспечить наличие товара на указанном в документе складе. Данная информация используется при составлении графика движения товаров. При проведении заказа в статусе "К обеспечению" контролируется наличие товаров на складе, указанных в документе. Программа предупреждает об отсутствии того или иного товара на складе.

Если потребность по заказу уже обеспечена: товар есть в наличии на складе и его можно отгрузить к тем датам, которые указаны для товаров в табличной части документа, то устанавливается статус "К выполнению".

Как установить дату отгрузки в заказе на внутреннее потребление?

Дата отгрузки в заказе на внутреннее потребление может быть задана вручную для каждого товара или рассчитана автоматически. Автоматический расчет даты отгрузки возможен только в том случае, если для заказа на внутреннее потребление установлен статус "К обеспечению". При расчете даты отгрузки учитывается та желаемая дата, к которой необходимо получить товар со склада.

Для расчета даты отгрузки необходимо выполнить следующие действия.

  • Установите статус "К обеспечению".
  • Укажите желаемую дату, к которой необходимо обеспечить наличие товара на складе.
  • Нажмите на кнопку "Состояние обеспечения".
  • В открывшейся форме установите флажки у всех строк (более подробно о работе с формой "Состояние обеспечения" см. справку к ней).
  • Нажмите на кнопку "Рассчитать дату отгрузки", далее "Для полной отгрузки" или "Для частичной отгрузки".
  • Дата отгрузки будет рассчитана с учетом возможности отгрузки товара с указанного склада.
  • При расчете даты отгрузки учитывается тот метод контроля остатков, который установлен на складе.

Примечание. Дата отгрузки может быть заполнена вручную для всех или нескольких выделенных позиций в документе. Для этого используется команды "Функции" - "Заполнить дату отгрузку для всех строк" или "Функции" - "Заполнить дату отгрузку для выделенных строк".

Важно. Если отгрузка всех товаров производится одной датой, то нужно установить флажок Отгружать одной датой и указать дату отгрузки вручную или выполнить команду Рассчитать дату отгрузки.

Как откорректировать заказ на внутреннее потребление?

В любой момент времени можно откорректировать количество в заказе на внутреннее потребление и перепровести документ. При этом будет автоматически откорректированы данные по обеспечению или выполнению заказа. Для любой позиции в заказе на внутреннее потребление можно установить признак "Отменено". После проведения документа эта информация отобразится в графике движения товаров, менеджер по логистике будет иметь оперативную информацию об изменении заказа на внутреннее потребление.

Как оформить заказ на внутреннее потребление в рамках конкретной сделки?

Для того что бы закупка и заказ товара прошли по конкретной сделке, необходимо в заказе на стрнице "Дополнительно", в поле "Сделка" указать нужную сделку.

Важно. Поле "Сделка" доступно в документе если в настройках параметров учета на странице "Финансы" установлены флажки "Обособленный учет" и "По сделкам".

Регистрация заказа на внутреннее потребление

Регистрация заказа производится при выполнении команды "Провести". Перед проведением заказа контролируется заполнение всех обязательных реквизитов: организация, склад, подразделение, дата отгрузки, ответственный и т.д. При проведении документа с установленным статусом "К выполнению" контролируется наличие товаров на складе в соответствии с тем методом контроля остатков, который установлен для склада.

Как в документе можно получить данные из терминала сбора данных?

Для того чтобы загрузить данные из терминала сбора данных в табличную часть "Товары" необходимо воспользоваться командой табличной части "Все действия" - "Загрузить товары из ТСД". Если терминал сбора данных подключен и настроены параметры его подключения, то табличная часть заполниться данными из терминала. В противном случае программа выдаст предупреждение. Если из теминала будет получен штрих код не зарегистрированый в информационной базе, то программа предложит зарегистрировать его и сопоставить с имеющимся в информационной базе товаром.

Как можно добавить данные в табличную часть "Товары" с помощью сканера?

Если сканер подключен и настроены параметры его подключения, то после считывания штрих кода в табличную часть будет добавлен товар соответствующий штрих коду. Если товар уже присутствует в табличной части, то будет увеличено его количество. Если в программе не зарегистрирован считываемый штрих код, то программа предложит зарегистрировать его и сопоставить имеющемуся в информационной базе товару.

Как найти в табличной части товар используя сканер штрих кода?

Для того что бы выполнить поиск товара в табличной части, необходимо воспользоваться командой табличной части "Все действия" - "Найти товары по штрих коду". Далее в открывшейся форме считать штрих код товара. Если товар со считанным штрих кодом присутствует в табличной части, то он будет выделен (на него будет установлен курсор). Если в программе не зарегистрирован считываемый штрих код, то программа предложит зарегистрировать его и сопоставить имеющемуся в информационной базе товару.
