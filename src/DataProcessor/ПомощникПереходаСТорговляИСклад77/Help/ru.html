Обработка "Помощник перехода с 1С:Торговля и склад 7.7"

С помощью этой обработки можно загрузить данные из конфигурации 1С:Торговля и склад 7.7.

Важно. С подробной информацией по переходу с конфигурации 1С:Торговля и склад на конфигурацию 1С:Управление торговлей можно ознакомиться в руководстве, которое можно найти в каталоге шаблонов конфигурации 1С:Управление торговлей.

Концепция переноса данных из конфигурации 1С:Торговля и склад 7.7 в конфигурацию 1С:Управление торговлей 11.0

Перенос данных предназначен для пользователей программы 1С:Торговля и склад, редакция 9.2 (ТиС), которые переходят на работу в программе 1С:Управление торговлей, редакция 11.0 (УТ). В результате переноса данных пользователи получают возможность начать работать в УТ с теми данными, с которыми они прежде работали в ТиС: это вся справочная информация, а также информация об остатках по всем разделам учета на определенную дату.

Перенос данных возможен для конфигураций:

  • Типовая конфигурация 1С:Торговля и склад, редакция 9.2 для 1С:Предприятия 7.7, версия 9.2.966 и более поздние версии.
  • Типовая конфигурация 1С:Управление торговлей, редакция 11 для 1С:Предприятия 8.2, версия 11.0.7.4 и более поздние версии.

Перенос данных разделен на два этапа. На первом этапе происходит перенос констант и справочников. На втором этапе происходит перенос остатков.

Каждый сеанс переноса данных включает в себя выгрузку определенной порции данных из ТиС и последующую загрузку этих данных в УТ. Допускается выполнение повторного переноса, за счет этого обеспечивается возможность синхронизации новых изменений данных, сделанных в ТиС.

Основные положения

Процесс переноса данных осуществляется последовательно. Можно выделить в процессе переноса два этапа, каждый из которых включает в себя перенос определенной части данных из ТиС в УТ.

Первый этап

На первом этапе процесса переноса данных выполняется автоматическая начальная настройка параметров в информационной базе УТ в соответствии с состоянием использования той или иной функциональности в информационной базе ТиС. Например, происходит подключение функциональности использования розничных продаж или использования дополнительных реквизитов объектов, если по данным информационной базы ТиС выявляется необходимость в их использовании.

На первом этапе процесса переноса данных также происходит перенос нормативно-справочной информации из информационной базы ТиС в информационную базу УТ.

После прохождения первого этапа предполагается проведение проверки настроек программы в информационной базе УТ. При необходимости производятся дополнительные настройки. Например, дополнительно можно подключить использование новой по сравнению с ТиС функциональности УТ.

После окончательной настройки программы проверяется правильность переноса нормативно-справочной информации. Количество и содержание проверок выбирается самостоятельно.

При проверке правильности переноса следует руководствоваться следующими принципами:

  • Проверка по приоритету данных. В первую очередь следует обратить внимание на правильность переноса информации о классификаторах, затем на правильность переноса справочной информации о структуре компании, затем на правильность переноса остальной справочной информации (по разделам). Среди прочей требующей проверки информации следует обратить внимание на ту информацию, которая наиболее востребована при работе в программе.

Пример. При проверке информации о клиентах в первую очередь имеет смысл обратить внимание на правильность переноса информации о тех клиентах, с которыми чаще всего приходится работать.
  • Проверка по репрезентативности данных. Для проверки данных можно выбрать не все, а несколько элементов, на которых могли проявиться различные особенности переноса данных.

Пример. При проверке контактной информации имеет смысл обратить внимание на правильность переноса контактной информации только у одного элемента справочника Контрагенты, а не у всех элементов.

Второй этап

На втором этапе процесса переноса в информационную базу УТ переносится информация о начальных остатках по всем разделам учета:

  • учет остатков товаров;
  • учет взаиморасчетов с покупателями;
  • учет взаиморасчетов с поставщиками;
  • учет взаиморасчетов с подотчетными лицами;
  • учет наличных денежных средств;
  • учет безналичных денежных средств.

Остатки можно не переносить в некоторых случаях, например, когда правильность данных учета в информационной базе ТиС вызывает сомнения. В этом случае достаточно будет перенести только элементы справочников, остатки следует ввести вручную.

После того как перенесены остатки, не рекомендуется изменять документы “задним числом”. В случае изменения документов задним числом остатки в информационных базах ТиС и УТ будут различаться. Для того, чтобы синхронизировать данные об остатках необходимо будет удалить все документы ввода начальных остатков в информационной базе УТ и выполнить повторный перенос информации об остатках.

После переноса данных рекомендуется проконтролировать совпадение остатков в информационных базах ТиС и УТ. Сравнение остатков осуществляется с помощью аналогичных отчетов ТиС и УТ.

После сравнения остатков можно приступить к ведению учета в УТ.

Выгрузка данных из информационной базы ТиС

Основным инструментом, который необходимо использовать при осуществлении перехода с ТиС на УТ, является внешняя обработка Помощник перехода на 1С:Управление торговлей 11. Данная обработка позволяет осуществить выгрузку данных из информационной базы ТиС с целью последующей загрузки этих данных в информационную базу УТ.

Важно. Подробная информация о выгрузке данных из информационной базы ТиС изложена в руководстве по переходу.

Загрузка данных в информационную базу УТ

Загрузка данных осуществляется с помощью данного обработки Помощник перехода с 1С:Торговля и склад 7.7, входящей в состав конфигурации УТ.

На первой странице помощника необходимо выбрать тот же файл, который был сформирован при выгрузке данных из ТиС. При нажатии на кнопку Далее помощник начнет загрузку данных.

По окончании загрузки в форме помощника будет отображен статус загрузки данных. Если данные загрузились с ошибками, в форме будет также отображен отчет о загрузке с описанием возникших ошибок. Если данные загрузились без ошибок, отчет о загрузке будет скрыт, но его можно отобразить путем нажатия на гиперссылку Отчет о загрузке.

При нажатии на копку Готово форма помощника будет закрыта. При нажатии на кнопку Назад помощник вернется на стартовую страницу.

Важно. Загрузку данных в УТ необходимо проводить по окончания каждого сеанса выгрузки данных из ТиС.
