Управление запасами

Основной целью подсистемы "Управление запасами" является эффективное управление запасами предприятия: удовлетворение потребностей внешних и внутренних потребителей в товарах предприятия.

Подсистема выполняет следующие функции.

  • Определяет правила управления и пополнения запасов по конкретным товарам на каждом складе, а также в целом по предприятию, на основе АВС/XYZ классификации товаров: заказ под заказ, заказ по точке перезаказа, объемно календарное планирование.
  • Рассчитывает необходимые товарные ограничения (минимальный запас, страховой запас, неприкосновенный запас) на основе анализа данных о продажах товаров за предыдущие периоды.
  • Позволяет прогнозировать спрос на товары с использованием различных источников спроса и учитывать прогнозируемый спрос при планировании закупок товаров.
  • Позволяет управлять потребностями в снабжении товаров, используя данные о методе управления запасами, прогнозируемом спросе, транспортных ограничениях поставщиков.
  • Позволяет автоматически контролировать обеспечение заказов клиентов, вовремя оформлять заказы поставщикам, заказы на внутреннее перемещение товаров исходя из потребности в товарах с учетом транспортных ограничений поставки товаров от поставщика, срока поставки с других складов и т.д.

Определение параметров управления запасами

Программа позволяет определить методы и параметры управления запасами, а также задать способы и параметры пополнения запасов на складах.

Определение методов и параметров управления запасами

Методы управления запасами, применяемые программой для определения оптимального запаса товаров на складах и заказа товаров, определяются программой автоматически на основании анализа данных о классификации товаров в соответствии с методом АВС/XYZ классификации. Методы управления запасами определяются для каждого склада и для каждого товара. При необходимости менеджер по закупкам может изменить рекомендуемый программой метод управления запасом.

В программе могут использоваться следующие методы управления запасами.

  • Объемно-календарное планирование.
  • Заказ под заказ.
  • Точка перезаказа (с фиксированным объемом заказа).
  • Точка перезаказа (с фиксированными интервалами между заказами).

Для каждого склада определяются те параметры, которые оказывают влияние на расчет оптимального запаса товаров на складе. Для склада определяются такие параметры, как график работы склада, принцип контроля товаров на складах. Для каждого склада определяется такой параметр, как допустимый уровень обслуживания покупателей при продаже товаров со склада.

На основании этих данных и анализа продаж товаров за определенный период автоматически рассчитываются необходимые товарные ограничения по каждой номенклатурной позиции на каждом складе: страховой запас, минимальный запас, нормативный запас. При расчете оптимального запаса на складе менеджер по закупкам может использовать рекомендуемые значения или ввести свои значения параметров запасов на складах.

Установка параметров для расчета товарных ограничений

Где определяется тот период, за который анализируются данные для расчета товарных ограничений?
  • В списке разделов выберите раздел "Запасы и закупки".
  • В панели действий, в группе "Сервис" выберите команду "Классификация номенклатуры".
  • Откроется форма "Классификация номенклатуры". Переключите форму в режим "Полные возможности".
  • Воспользуйтесь командой "Настройка", в группе команд "Товарные ограничения".

Расчет товарных ограничений

Расчет товарных ограничений (минимального, страхового и нормативного запаса) по каждому товару производится автоматически с помощью регламентного задания "Расчет товарных ограничений". Если использование регламентного задания по разным причинам невозможно, то автоматический расчет товарных ограничений можно запустить вручную. После расчета менеджер по закупкам в случае необходимости может вручную откорректировать результаты автоматического расчета товарных ограничений для конкретной номенклатурной позиции.

Как открыть форму для выполнения расчета товарных ограничений?
  • В списке разделов выберите раздел "Запасы и закупки".
  • В панели действий, в группе "Сервис" выберите команду "Классификация номенклатуры".

Как посмотреть результаты расчета товарных ограничений (минимальный запас, страховой запас, нормативный запас товара)?

Информацию о рассчитанных товарных ограничениях можно посмотреть в отчете "ABC/XYZ - анализ номенклатуры". Рассчитанные товарные ограничения будут показаны по каждому товару в графе "Рекомендуемый". Отчет может быть сформирован как в целом по предприятию, так и по конкретному складу. Регламентным заданием товарные ограничения рассчитываются про каждому складу. Расчет товарных ограничений в целом по предприятию определяется, как суммарное значение показателей по каждому складу.

Для просмотра информации о товарных ограничениях по конкретному товару в панели навигации карточки товара выберите пункт меню "Товарные ограничения". В таблице будут показаны результаты расчета товарных ограничений по каждому складу. Рассчитанные товарные ограничения показываются во второй колонке (рекомендуемый). В первой колонке показываются используемые значения. Используемые значения могут отличаться от рекомендуемых (после ручного изменения данных менеджером по закупкам).

Товарные ограничения рассчитываются на основании среднего количества товаров, проданных за день. Среднее количество товаров за день рассчитывается на основании анализа данных о продажах товаров на каждом складе за указанный период расчета. При расчете этого показателя учитываются только те дни, когда товар реально продавался со склада.

  • Страховой запас. Рассчитывается как среднеквадратичное отклонение от среднего количества товара, проданного за день.
  • Минимальный запас. Рассчитывается как среднедневное потребление товаров со склада в течении анализируемого периода, умноженное на длительность доставки в днях, определенную в транспортных ограничениях.
  • Нормативный запас. Рассчитывается по следующей формуле: Минимальный запас+ (Страховой запас* Коэффициент уровня обслуживания покупателей) Коэффициент уровня обслуживания покупателей задается для каждого склада ("Уровень обслуживания").
  • Максимальный запас. Не рассчитывается, он может быть установлен менеджером по управлению запасами вручную.

Примечание. Товарные ограничения рассчитываются с точностью до трех знаков после запятой. При использовании этих значений в обработке "Управление запасами" они округляются до целого числа.

Как определяются методы управления запасов?

В соответствии с проведенной ABC/ XYZ классификацией товаров методы управления запасами устанавливаются автоматически для каждого склада и каждой позиции номенклатуры. Для анализа используются следующие показатели: для АВС классификации используется показатель "Выручка", а для XYZ - классификации "Количество проданных товаров". Используется следующая методика автоматического расчета методов управления запасами.

  • Метод "Объемно-календарное планирование" устанавливается для товаров, которые относятся к группам АX, AY, BX, BY. Эти товары стабильно продаются и имеют высокий товарооборот, то есть приносят торговому предприятию высокий доход. При расчете запаса товаров по данному методу выбирается максимальный показатель из следующий показателей: нормативный запас, данные планов продаж и внутреннего потребления, потребности по заказам.
  • Метод "Точка перезаказа (с фиксированным интервалом времени между заказами)" устанавливается для товаров, которые относятся к группе CX (товары продаются регулярно, но имеют небольшой товарооборот). При расчете запаса товаров по данному методу выбирается максимальный показатель из следующий показателей: нормативный запас и потребности по заказам. При этом учитываются также даты, когда поставщик может поставлять товар (транспортные ограничения).
  • Метод "Точка перезаказа (с фиксированным размером заказа)" устанавливается для товаров, которые относятся к группе CY (товары продаются нерегулярно и имеют небольшой товарооборот). При расчете запаса товаров по данному методу выбирается максимальный показатель из следующий показателей: нормативный запас и потребности по заказам. При этом учитываются также товарные ограничения. Объем заказа определяется максимальным размером запаса.
  • Метод "Заказ под заказ" устанавливается для товаров, которые относятся к группам AZ,BZ, CZ. По таким товарам на складе не хранится запас, товар заказывается у поставщика только при возникновении потребности по заказам (оформлении заказа клиента, заказа на перемещение товаров и т.д.).

Как посмотреть какой метод управления запасами установлен для товаров на складах?

Данную информацию можно посмотреть в отчете "ABC/XYZ - анализ номенклатуры". Отчет вызывается из раздела "Запасы и закупки".

Метод управления запасами определяется по каждому товару на конкретном складе. Для просмотра установленного метода управления запасами выберите вариант отчета "ABC/XYZ - анализ номенклатуры (запасы по складам)". Метод управления запасами определяется программой автоматически на основании данных об АВС/XYZ - классификации товаров и может быть изменен пользователем вручную. Если метод управления запасами отличается от рекомендуемого, то он помечается в отчете красным цветом.

Важно. Если классификация товаров не выполнена, то для таких товаров устанавливается рекомендуемый метод управления запасами "Заказ под заказ".

Определение способов и параметров пополнения запасов

Кроме товарных ограничений для каждого склада и номенклатурной позиции можно установить транспортные ограничения. Например, можно указать, что запасы товаров на складе пополняются только путем перемещения с другого склада или запасы определенного товара закупаются у конкретного поставщика.

Установка транспортных ограничений

Транспортные ограничения могут быть заданы для конкретного склада и конкретных позиций номенклатуры.
Предусмотрена также возможность задавать обобщенные транспортные ограничения: любой товар может поступает на склад путем перемещения с другого склада, товар на склад может поступать от любых поставщиков.

Важно. Установленные транспортные ограничения используются в обработке "Управление запасами". Если транспортное ограничение задано, то именно оно будет использоваться в качестве источника пополнения запасов. При этом будет учитываться указанный срок транспортировки. Если товар от поставщика доставляется в течении недели, то рекомендации к оформлению заказа поставщику будут сформированы на неделю раньше планируемой потребности. Если транспортные ограничения не заданы, то считается, что товар может быть закуплен у любого поставщика и товар может быть поставлен в течении дня. Потребность в оформлении заказа поставщику будет показана в день ее возникновения, будет показан список всех поставщиков, у которых товар может быть закуплен.

Перед установкой транспортных ограничений необходимо ввести информацию о тех способах пополнения запасов, которые используются в торговом предприятии.

Ввод информации о способах пополнения запасов

Как открыть справочник "Способы пополнения запасов"?
  • В списке разделов выберите раздел "Запасы и закупки".
  • В панели навигации раздела выберите пункт "Способы пополнения запаса".

Ввод информации о транспортных ограничениях

Информация о транспортных оганичениях может быть введена для конкретного склада (из карточки склада) или для конкретной позиции номенклатуры (из карточки позиции номенклатуры).

Управление потребностями в снабжении

Определение и распределение потребностей в снабжении

Программа позволяет контролировать запасы товаров на каждом складе. Для каждого склада может быть установлен свой метод контроля остатков, который определяется спецификой работы конкретного склада. Для оперативного контроля запасов товаров на складах, контроля ожидаемых отгрузок и поступлений товаров используются отчеты "Анализ доступности товаров на складах" и "Товарный календарь".

Программа позволяет отслеживать дефицит товара в соответствии с рассчитанным для товара и склада нормативным запасом и оформленными заказами клиентов. Для тех товаров, которые хорошо и стабильно продаются (используют метод объемно-календарного планирования) при анализе потребностей учитывается планируемый и прогнозируемый спрос на товары.

Для контроля запасов и отслеживания дефицита товаров используется обработка "Управление запасами". С помощью этой обработки менеджер может не только контролировать запасы товаров, но и оформлять заказы на перемещение товаров, заказы на сборку, заказы поставщикам. Менеджер по закупкам может внести в программу свои рекомендации (транспортные ограничения) по управлению запасами: на какой склад поступают товары от конкретного поставщика и с какой задержкой они поставляются, в какой торговый зал товар поступает только путем перемещения с оптового склада и т.д. Все эти рекомендации менеджера по закупкам будут учтены при автоматическом формировании заказов на обеспечение (заказов на внутреннее потребление, заказов на сборку, заказов поставщикам).

Оперативный контроль запасов товаров на складах

В процессе оформления документов отгрузки программа контролирует запасы товаров на складах. Для каждого склада может быть установлен свой вариант контроля остатков в зависмости от специфики работы склада. При оформлении документов, которые являются распоряжениями на отгрузку товаров со склада автоматически рассчитывается предполагаемая дата отгрузки товаров, которая фиксируется в графике движения товаров. В любой момент времени пользователь может посмотреть информацию о свободных остатках товаров и о планируемых поступлениях и отгрузках товаров по заказам.

Просмотр информации о запасах товаров

В оперативной работе для просмотра информации о текущих запасах товаров пользователи могут использовать отчеты "Анализ доступности товаров на складах" и "Товарный календарь".

  • Для вызова отчетов в списке разделов выберите раздел "Запасы и закупки".
  • В панели действий раздела в группе "Отчеты" выберите нужный отчет "Анализ доступности товаров на складах" или "Товарный календарь".

Как посмотреть сколько товаров можно сегодня отгрузить?

Эту информацию можно получить с помощью отчета "Анализ доступности товаров на складах. "

В отчете отображается информация о текущем остатке на складе ("В наличии"), о количестве товаров, которое планируется отгрузить со склада ("Резерв"), о количестве свободного остатка ("Свободный остаток").

  • В наличии - это то количество товара, которое реально находится на складе.
  • Резерв - количество зарезервированного товара. В количество зарезервированного товара входят те товары, которые запланированы к отгрузке и те товары, которые были зарезервированы из текущего остатка при оформлении заказа клиента в статусе "К обеспечению".
  • Свободно - количество товаров, которое может быть отгружено на текущую дату. Количество свободного товара рассчитывается по формуле Свободно = "В наличии" - "Резерв"

Важно. Резервирование товаров на складе по заказу клиента зависит от установленного варианта контроля остатков на складе.
Если на складе установлен вариант контроля "Остатки с учетом резерва", то товар резервируется из текущего остатка товаров на складах на дату оформления заказа клиента, независимо от того, какая дата отгрузки установлена в заказе клиента. Если на складе установлен вариант контроля "Остатки с учетом графика", то программа сначала рассматривает вариант размещения товара в предстоящих поступлениях. Если товар не может быть размещен в предстоящих поступлениях и имеется в наличии на складе на дату оформления заказа клиента, то товар резервируется из текущего остатка товаров на складе на дату оформления заказа клиента.

Как посмотреть план поступлений и отгрузок товаров?

План поступлений и отгрузок по товарам можно посмотреть в отчете "Товарный календарь".

  • Установите в отчете ту конечную дату, на которую необходимо посмотреть план поступлений и отгрузок товаров.
  • В отчете будет показан план поступлений и отгрузок, детализированный по датам планируемого поступления и расхода товаров с момента формирования отчета (текущей даты) до указанной даты окончания периода планирования ("Дата окончания").
  • В поле "Приход" показывается то количество товаров, которое запланировано к поступлению. Эта информация формируется на основании количества товаров и даты поступления, указанной в заказах поставщиков в статусе "К поступлению" и во внутренних заказах (заказах на перемещение, заказах на сборку) в статусе "К обеспечению".
  • В поле "Расход" показывается количество товаров, которое запланировано к отгрузке. Эта информация формируется на основании количества товаров и даты отгрузки, указанной в заказах клиентов в статусе "К обеспечению" и во внутренних заказах (заказах на внутренне потребление, заказазах на перемещение, заказах на сборку) в статусе "К обеспечению".

Управление запасами

С помощью обработки "Управление запасами" менеджер по закупкам может контролировать запасы товаров на складах, отслеживать дефицит товаров, автоматически формировать заказы поставщикам, а также заказы на перемещение товаров, заказы на сборку (разборку) товаров.

Предлагается следующий порядок работы с обработкой "Управление запасами".

  • Менеджер по логистике открывает обработку "Управление запасами".
  • Просматривает информацию о просроченных отгрузках и поступлениях товаров. Корректирует даты отгрузки и поступления товаров.
  • Просматривает информацию о тех товарах, по которым есть дефицит на указанный им период планирования.
  • Используя рекомендации программы, автоматически формирует заказы поставщикам, заказы на перемещение товаров, заказы на сборку (разборку).

Оформление заказов на пополнение товарных запасов (заказов поставщиков, заказов на перемещение, сборку и т.д.)

Обработка "Управление запасами" позволяет автоматически формировать заказы на поставку товаров на указанный в обработке период планирования. Заказы формируются на то количество товаров, которое указано в графе "К заказу". При расчете потребностей в товарах анализируется какой метод управления запасами применяется при расчете запаса товаров для конкретной номенклатурной позиции и конкретного склада.

  • При методе управления запасами "Объемно-календарное планирование" для расчета потребностей используется максимальное значение из нормативного запаса, прогнозируемого спроса и количества товаров, заказанного клиентами на период планирования с учетом уже оформленных заказов поставщиков.
  • При методе управления запасами "Точка перезаказа" для расчета потребностей используется максимальное значение из нормативного запаса и количества товаров, заказанного клиентами на период планирования с учетом уже оформленных заказов поставщиков.
  • При методе управления запасами "Заказ под заказ" расчет потребностей производится только на основании данных о неудовлетворенном спросе по заказам клиентов.

Программа выдает рекомендации о том, какой товар необходимо заказать. При этом в качестве рекомендаций по устранению дефицита товаров может быть не только заказ у конкретного поставщика, но и заказ на перемещение товаров с другого склада или заказ на сборку (разборку). При формировании заказов учитываются те траспортные ограничения, которые были заданы для склада или конкретного товара.

Пример. В качестве транспортного ограничения задано, что товары на розничный склад должны быть оформлены путем перемещения товаров с оптового склада. В этом случае в качестве рекомендации для пополнения товаров на розничном складе будет автоматически сформирован "Заказ на перемещение" с оптового склада на то количество товаров, которое имеется в дефиците на розничном складе.

Оформление заказов на перемещение

Заказы на перемещение оформляются при возникновении в потребности перемещения товаров между складами предприятия. Заказы на перемещение могут быть сформированы автоматически с помощью обработки "Управление запасами", если возникает потребность в пополнении запаса товаров путем пемещения с другого склада (определен способ пополнение запаса -перемещение). Заказы на перемещение могут быть оформлены и вручную при возникновении необходимости перемещения товаров со склада на склад.

Как открыть список заказов на перемещение товаров?
  • В списке разделов выберите раздел "Запасы и закупки".
  • В панели навигации раздела выберите пункт "Заказы на перемещение".

Оформление заказов на сборку (разборку)

Заказы на сборку (разборку) формируются при возникновении потребности сборки комплектов из комплектующих или разборки готового комплекта на комплектующие. Заказы на сборку (разборку) могут быть сформировании автоматически с помощью обработки "Управление запасами" при возникновении потребности в сборке (разборке) (способ поставки - сборка/разборка). Предусмотрена также возможность ручного оформления заказов на сборку(разборку).

Как открыть список заказов на сборку/разборку товаров?

  • В списке разделов выберите раздел "Запасы и закупки" .
  • В панели навигации раздела выберите пункт "Заказы на сборку/разборку".

Контроль обеспечения заказов

Для контроля обеспечения заказанных товаров предназначена обработка "Состояние обеспечения заказов". Контролировать состояние обеспечения можно по тем заказам, по которым планируется отгружать товар: Заказ клиента, Заказ на перемещение, Заказ на сборку, Заказ на внутреннее потребление. Также контролируется обеспечение по тем заявкам на возврат товаров, по которым предусмотрена замена товаров.

Как открыть обработку для контроля обеспечения заказов?
  • В списке разделов выберите раздел "Запасы и закупки".
  • В панели навигации раздела выберите пункт "Состояние обеспечения заказов".

Планирование потребностей в запасах

Программа позволяет прогнозировать потребность в запасах товаров, используя для составления прогноза различные источники данных. В качестве источников данных могут быть использованы любые данные, хранящиеся в информационной базе. Например, объем продаж за предыдущие периоды продаж, объем закупок, текущий остаток на складе и т.д. Список источников данных, которые используются для планирования потребностей в запасах, может пополняться пользователем произвольным образом.

При расчете планируемых потребностей в запасах пользователь может применять различные правила преобразования источников данных. Например, при расчете планируемых запасов учитывать максимальный объем продаж в течение квартала, исключая при этом текущие складские остатки товаров и планируемые поставки товаров в течении периода планирования потребностей. Правила преобразования источников задаются и хранятся в документе "План продаж и внутреннего потребления".

Рассчитанный планируемый объем потребностей фиксируется в документе "План продаж и внутреннего потребления". В дальнейшем эта информация используется в обработке "Управление запасами" для расчета оптимального запаса товаров на складах и оформления документов поставки товаров на склад.

Важно. Рассчитанный планируемый объем потребностей используется только при расчете оптимального запаса товаров на складах для тех товаров, для которых установлен метод управления запасами - "Объемно-календарное планирование".

Настройка правил формирования планов-прогнозов

Для настройки правил формирования планов-прогнозов используются источники данных. В качестве источника данных может использоваться любая информация, зарегистрированная в информационной базе (текущие остатки товаров, количество товаров, ожидаемых к поступлению и т.д.). Для настройки источников данных используется система компоновки данных. Программа предоставляет несколько готовых шаблонов схемы компоновки данных, с помощью которых пользователь может быстро создавать новые источники данных.

В дальнейшем эти источники данных используются для заполнения план-прогнозов. Для каждого план-прогноза можно задать свою схему преобразования источника данных. Например, выбрать максимальное значение объема продаж за разные месяцы продаж и учесть текущий остаток товаров и планируемые поставки товаров по заказам поставщиков.

Как открыть список источников данных?
  • В панели разделов выберите раздел "Запасы и закупки".
  • В панели навигации раздела выберите пункт "Источники данных планирования".

Формирование планов-прогнозов продаж и потребления

Планы-прогнозы продаж и внутреннего потребления формируются с помощью документа "План продаж и внутреннего потребления". Документ "План продаж и внутреннего потребления" может быть заполнен как вручную путем добавления информации о товарах в табличную часть документа, так и автоматически. Автоматическое заполнение документа производится на основе анализа данных источников, указанных в плане и правил преобразования этих источников. Список источников и правил преобразования сохраняются в документе.

Как открыть список документов "План продаж и внутреннего потребления"?
  • В панели разделов выберите раздел "Запасы и закупки".
  • В панели навигации раздела выберите пункт "Планы продаж и внутреннего потребления".
