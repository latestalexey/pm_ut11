Какую информацию необходимо ввести в график оплаты?

Для каждого графика оплаты указывается наименование, календарь, в соответствии с которым заполняются даты оплаты в документах, а также список этапов оплаты.

Для каких целей используется наименование в графике оплаты?

Наименование используется для дальнейшей идентификации графика оплаты при заполнении его в соглашениях с клиентами и документах продажи. В качестве наименования может быть введена любая текстовая строка. Например, "Полная предоплата наличными" или "Предоплата 30%, отсрочка платежа на месяц".

Можно ли при расчете дат оплаты учитывать выходные дни?

Учет выходных дней производится в соответствии с тем календарем, который указан в графике оплаты. Для каждого графика оплаты может быть указан свой календарь, по которому будут учитываться рабочие и выходные дни при расчете дат оплаты в соответствии с установленными значениями в графике оплаты. Таким образом можно настроить различные варианты учета выходных дней для различных графиков оплаты. Например, в графике оплаты с авансовыми платежами выходные дни учитываются, а при использовании графика оплата по кредитным отношениям, кредит предоставляется на месяц, без учета выходных дней.

Как заполнить список этапов?

Список этапов заполняется в табличной части этапов оплаты. График оплаты может состоять и из одного этапа (например, Полная (100%) предоплата).

  • Для ввода нового этапа нажмите на кнопку "Добавить".
  • Для каждого этапа заполните информацию о варианте платежа (предоплата, аванс, кредит), проценте платежа, форме оплаты и отсрочке платежа.

При заполнении графика оплаты необходимо соблюдать следующие правила.

  • Процент платежа по всем этапам должен быть 100%.
  • Строки со значениями "Предоплата" и"Аванс" в поле "Вариант оплаты" должны располагаться раньше строк со значением "Кредит".
  • Количество дней отсрочки должно идти по возрастанию для каждого варианта оплаты.

Какие варианты оплат могут быть зарегистрированы для этапа оплаты?

Предусмотрены три варианта оплаты, которые можно выбрать из выпадающего списка.

  • Аванс (до обеспечения). Клиент должен оплатить заказ до момента обеспечения товаров на складе. То есть заказ недостающих товаров у поставщика, сборка товаров на складе, резервирование товаров, будет производиться только после того, как клиент оплатил заказ.
  • Предоплата (до отгрузки). Клиент должен оплатить заказ до момента отгрузки товара. При этом заказ может быть передан для обеспечения до момента фиксации факта оплаты товаров клиентом.
  • Кредит (после отгрузки). Клиенту предоставляется кредит и оплата должна производиться после отгрузки товара.

Как указать процент платежа?

Процент платежа указывается в поле "% платежа". Процент указывается от общей суммы, указанной в документе "Заказ клиента". Процент платежа необходим для расчета суммы платежа по каждому этапу, при заполнении информации об этапах оплаты в заказе клиента.

Как указать количество дней допустимой отсрочки платежа?

Количество дней отсрочки платежа указывается в поле "Отсрочка". Количество дней отсрочки платежи всегда считается от даты, указанной в документе продажи.

  • Для аванса и предоплаты - это количество дней, когда должен быть оплачен заказ. Например, заказ выписан 10 января, отсрочка 3 дня, срок оплаты (если учитываются и выходные дни) - 13 января.
  • Для кредита - срок, на который предоставлен кредит. Например, отгрузка произведена 10 января, отсрочка - 30 дней, срок оплаты (если учитываются и выходные дни) -10 февраля.

При расчете дней отсрочки платежа можно учитывать или не учитывать выходные дни. Это определяется календарем, указанным в графике оплаты.

Обязательно ли указание формы оплаты при заполнении этапа оплаты?

Указание планируемой формы оплаты не является обязательной при вводе этапа оплаты. Указанная планируемая форма оплаты клиентом (наличная, безналичная или платежная карта) используется в качестве значения по умолчанию при заполнении этапов оплаты в заказе клиента. Эта информация может быть уточнена в заказе и в дальнейшем использована при планировании платежей и для составления платежного календаря.
