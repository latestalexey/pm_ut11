Работа со списком типовых соглашений

Отбор соглашений

Как сделать так, чтобы в списке показывались только те соглашения, за которые отвечает менеджер, работающий с программой?
  • В списке соглашений в поле "Менеджер" выберите значение "Мои документы".
  • В списке будут показаны только те соглашения, для которых менеджер, работающий в программе (текущий пользователь) установлен в качестве ответственного менеджера.

Как отобрать в списке только действующие соглашения?
  • В поле "Текущее состояние" выберите значение "Действует".
  • В поле "Срок действия" выберите значение "Непросроченные".

Как отобрать соглашения, по которым завтра истекает срок действия?
  • В поле "Текущее состояние" выберите значение "Действует".
  • В поле "Срок действия" выберите значение "Истекающие на дату" и укажите завтрашнюю дату.

Будут показаны те соглашения, у которых завтра истекает срок действия, то есть те соглашения, у которых в качестве конечной даты, указанной в периоде действия соглашения, указана дата больше или равна указанной (завтрашней) дате.

Как отобрать те соглашения, у которых завтра истекает срок согласования?
  • В поле "Текущее состояние" выберите значение "Ожидается согласование".
  • В поле "Срок действия" выберите значение "Истекающие на дату" и укажите дату, равную завтрашней дате.

Будут показаны те соглашения, у которых установлен статус "Не согласовано" и дата согласования меньше или равна указанной (завтрашней) дате.

Важно. В качестве даты согласования используется дата, которая указана в качестве начальной даты в периоде действия соглашения. Соглашение должно быть согласовано до момента начала его действия.

Почему соглашения в списке окрашены в разные цвета, что это означает?

В списке соглашений используется следующая градация цветов.

  • Кирпичным цветом помечаются все просроченные соглашения (отмечается поле списка "Текущее состояние"), то есть соглашения, у которых истек срок действия или срок согласования.
  • Бледно-желтым цветом помечаются соглашения, в которых определены регулярные условия поставок товаров.
  • Бледно-серым цветом помечаются закрытые соглашения.

Примечание. Изменить цвет фона (текста) или условия, по которым помечаются соглашения в списке, а также добавить свои условия, по которым будет раскрашен список, можно в режиме 1С:Предприятия, используя стандартные возможности платформы по настройке условного оформления списка.

Как создать новое типовое соглашение?

Новое типовое соглашение создается из списка типовых соглашений.

  • В списке типовых соглашений нажмите на кнопку "Создать".
  • Будет создано новое типовое соглашение, для которого необходимо определить период действия и правила продаж.
  • Новое типовое соглашение создается со статусом "Не согласовано".
