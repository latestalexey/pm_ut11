Заполнение информации в карточке вида цены

Что указывается в качестве наименования вида цены?

В качестве наименования можно использовать любое название цены: оптовая, дилерская и т.д. Фактически - это название колонки прайс-листа, который применяется на предприятии.

Для каких целей используется идентификатор вида цены?

Идентификатор для вида цены необходимо задавать для того, чтобы в дальнейшем можно было бы использовать данный вид цен в формулах расчета цен. Идентификатор может быть введен вручную или заполнен в соответствии с наименованием цены. Для формирования идентификатора на основании наименования цены, необходимо нажать на стрелку, рядом с полем ввода. После записи вида цены информация об идентификаторе не редактируется.

Как ввести валюту, в которой будут регистрироваться цены данного вида?

В качестве валюты вида цены можно выбрать любую валюту, которая зарегистрирована в справочнике "Валюта". Цены данного вида могут быть зарегистрированы только в той валюте, которая указана для данного вида цен.

Как используется флаг "Цена включает НДС"?

Если флаг установлен, то по данному виду цен значения цены должны указываться с учетом НДС. Если флаг не установлен, то цены указываются без учета НДС.

Как ограничить список тех видов цен, которые будут использоваться при продаже товаров?

Если данный вид цен будет использоваться при продаже товаров, то установите флаг "Использовать при продаже". Если данный вид цен является вспомогательными (например, служит основанием для расчета цен продажи ) и не должен использоваться при продаже товаров, то снимите флаг "Использовать при продаже".

Какие способы задания цены предусмотрены в программе?

Предусмотрены три способа задания цены:

  • Задавать вручную . Цены задаются вручную и могут быть использованы для расчета других цен.
  • Заполнять по данным ИБ. Цены автоматически рассчитываются на основании данных, хранящихся в информационной базе. Например, цена рассчитывается, как максимальная цена конкурента или как себестоимость товара. Эти цены могут быть использованы для расчета других видов цен.
  • Рассчитывать по другим видам цен. Цены рассчитываются по заданным формулам на основании других цен. В качестве базовых цен для расчета могут применяться любые виды цен (заданные вручную, заполненные по данным ИБ, рассчитанные на основании других видов цен).
  • Заполнять по данным ИБ при поступлении. Цены автоматически рассчитываются на основании данных, хранящихся в информационной базе. При вводе документа Установка цен номенклатуры на основании документа Поступление товаров и услуг, такие виды цен будут выбраны автоматически (в таблице видов устанавливаемых цен такие виды будут автоматически отмечены флажками).

По умолчанию устанавливается способ задания цены "Задавать вручную". Если необходимо изменить способ задания цены, то установите флаг в соответствующем поле. Для способов задания цены, отличных от способа "Задавать вручную" необходимо настроить правила расчета.

Настройка правил расчета цены

Как ввести правила расчета для тех видов цен, которые рассчитываются на основании данных в информационной базе ( себестоимость товаров, минимальная цена конкурентов и т.д). ?

В этом случае используется способы задания цены "Заполнять по данным ИБ" или "Заполнять по данным ИБ при поступлении". Данные из информационной базы формируются при выполнении запроса к информационной базе и задаются с помощью схемы компоновки данных (СКД). При выборе указанных способов задания цен в форме становится доступно поле "Схема компоновки данных", в котором можно указать предопределенные значения СКД. Формирование схемы компоновки данных возможно только в режиме "толстого" клиента.

В СКД для расчета цены могут применяться параметры и отборы. По команде "Редактировать" можно перейти к форме настройки СКД, где можно установить отбор и задать значение параметров для текущего вида цены. В форме настройки СКД во команде "Редактировать схему компоновки" можно изменить любую СКД, но при этом изменения текущей СКД будут сохранены в предопределенной СКД "Произвольный" (т.е. предопределенные СКД изменить нельзя, а для создания собственной СКД необходимо пользоваться предопределенной СКД "Произвольный").

Схема компоновки данных должна содержать только один набор данных, который должен обязательно содержать следующие поля:

  • Номенклатура, типа СправочникСсылка.Номенклатура.
  • Характеристика, типа СправочникСсылка.ХарактеристикиНоменклатуры.
  • Упаковка, типа СправочникСсылка.УпаковкиНоменклатуры.
  • Цена, типа число.
  • Коэффициент, типа число.
  • Валюта, типа СправочникСсылка.Валюты.

Структуру выводимых группировок и полей настраивать не нужно. Она будет игнорироваться.

Можно ли использовать уже готовые варианты схем запросов для расчета цен?

Поле "Схема компоновки данных" можно заполнить предопределенным значением схемы компоновки данных (СКД). Установленная СКД будет применяться для вида цены при расчете цен (при заполнении) документа "Установка цен номенклатуры". Доступны следующие предопределенные значения СКД:

  • "Цена поступления" – для номенклатуры будет отобрана последняя цена закупки.
  • "Себестоимость без дополнительных расходов" – в качестве цены будет использоваться рассчитанная на дату установки цены себестоимость товара.
  • "Себестоимость с дополнительными расходами" – в качестве цены будет использоваться рассчитанная на дату установки цены себестоимость товара, с дополнительными расходами.
  • "Максимальные цены поставщиков" – будет отобрана максимальная цена на товар (зафиксированная в документе Регистрация цен поставщика) среди всех поставщиков на дату установки цены. В качестве поставщиков будут использоваться партнеры с установленным флажком Поставщик.
  • "Средние цены поставщиков" – в документ будет отобрана средняя цена на товар, зафиксированная в документе Регистрация цен поставщика, среди поставщиков.
  • "Минимальные цены поставщиков" – в документ будет отобрана минимальная цена на товар, зафиксированная в документе Регистрация цен поставщика, среди поставщиков.
  • "Минимальные цены конкурентов" – будет отобрана максимальная цена на товар (зафиксированная в документе Регистрация цен поставщика) среди всех конкурентов на дату установки цены. В качестве конкурентов будут использоваться партнеры с установленным флажком Конкурент.
  • "Средние цены конкурентов" – в документ будет отобрана средняя цена на товар, зафиксированная в документе Регистрация цен поставщика, среди конкурентов.
  • "Максимальные цены конкурентов" – в документ будет отобрана минимальная цена на товар, зафиксированная в документе Регистрация цен поставщика, среди конкурентов.
  • "Произвольный" – если выбрано это значение, то для вида цены необходимо создать собственную СКД.

Как настроить правила расчета для цен, которые могут рассчитываться на основании других цен?

  • Для таких видов цен в поле "Способ задания цены"необходимо выбрать значение "Рассчитывать по другим видам цен".
  • В поле "Алгоритм расчета цены"с помощью редактора формул задать формулу расчета.

Как указать общую формулу расчета для всех ценовых групп товаров?
  • При установке флага "Рассчитывать по другим видам цен" будет показана табличная часть "Правила расчета".
  • В табличной части появится запись "Все ценовые группы".
  • В поле "Формула" введите формулу для расчета указанного вида цен для всех ценовых групп товаров.

Как ввести формулу расчета, если при расчете цен на разные ценовые группы используются разные наценки?

Формулы расчета можно детализировать до ценовых групп.

  • В табличной части "Правила расчета" нажмите пиктограмму "Добавить" или нажмите кнопку INS.
  • В появившейся новой строке выберите из списка "Ценовые группы" ту ценовую группу, для которой формула расчета отличается от общей формулы расчета.
  • Введите формулу расчета для конкретной ценовой группы.

Как ввести формулу расчета?

Формулу расчета можно заполнить вручную в поле или колонке табличной части "Алгоритм расчета цены" или ввести формулу с помощью конструктора формул.

Как написать формулу расчета вручную?

Формула расчета цены может быть введена вручную путем набора текстовой информации в соответствующем поле/колонке. При ручном вводе следует соблюдать правила написания выражений, которые предусмотрены для встроенного языка 1С:Предприятие. Для проверки правильности написания выражений следует использовать кнопку "Проверить формулы".

Пример. Дилерская цена рассчитывается как оптовая цена минус 40%. Формула для расчета должна быть написана следующим образом:
[Оптовая] * 0.4. В квадратных скобках указывается идентификатор вида цены.

Настройка правил округления

Для каждого вида цен можно указать правила округления цены. Для каждого диапазона значений (от 0 до 100 руб., от 100 руб. до 100 руб. и т.д.) задается точность округления цены после применения всех скидок и наценок (до 1 копейки, до 5 копеек, до рубля и т.д.). Также можно создать "психологическую скидку", т.е. правило анти-округления, которое применяется после округления и позволяет получить "некруглую" конечную цену. "Психологическая скидка" применяется для создания у покупателя впечатления более низкой цены.

Кроме этого, может быть установлен признак округления в большую сторону. При установке данного признака округление всегда будет происходить в большую сторону, а не по арифметическим правилам.

Как настроить правила округления для всех цен данного вида?
  • В форме "Вид цены" перейдите на закладку "Правила округления цены".
  • Нажмите на кнопку "Добавить" .
  • Появится строка в которой написано "Все цены".
  • В графе "Точность округления" укажите точность округления.
  • В графе "Психологическое округление" укажите последнюю цифру для психологического округления. При настройке психологического округления можно использовать конструктор (кнопка "Конструктор психологического округления").

Как настроить различные правила округления для разных диапазонов цен?
  • Откройте закладку "Правила округления цены".
  • Нажмите на кнопку "Добавить".
  • В поле "Нижняя граница" введите значение нижней границы диапазона, например 1000. Границы диапазона регистрируются в той валюте, который задан для вида цены.
  • В поле "Ценовой диапазон" автоматически сформируется надпись "Свыше 1000 рублей".
  • Введите для этого диапазона точность округления и последнюю цифру психологического округления.
  • Проделайте аналогичные действия для ввода следующего диапазона.

Пример. На предприятии используется следующий принцип округления. Для товаров, которые имеют цены от О до 100 рублей, цены округляются до десяти копеек.
Для товаров от 100 до 10000 рублей цены округляются до 1 рубля.
Для товаров от 10000 до 100000 рублей цены округляются до 10 рублей.
Для товаров выше 100000 рублей цены округляются с использованием психологического округления. Цена 100000 формируется как 99999.
  • Добавляем в список следующие диапазоны:
  • Нижняя граница - 0 рублей, точность округления 0, 1.
  • Нижняя граница - 100 рублей , точность округления - 1
  • Нижняя граница - 10000, точность округления - 10.
  • Нижняя граница -100000, точность округления 10, Психологическое округление - 9.

Настройка порога срабатывания

Для каждого вида цен можно указать порог срабатывания. Порог срабатывания - это процент отклонения вновь назначаемых цен от ранее зарегистрированных цен. Для каждой ценовой группы может быть назначен свой порог срабатывания. Порог срабатывания применяется в документе "Установка цен номенклатуры" при автоматическом расчете отпускных цен на основании цен поступления.

Пример. В виде цен Цена поступления для ценовой группы "Вентиляторы" установлен порог срабатывания равный 1%. Это означает, что новые цены поставки на товары этой ценовой группы не будут регистрироваться, если значение новой цены поставки отличается от старой цены менее чем на 1%.

Как указать одинаковый порог срабатывания для всех ценовых групп?
  • В форме "Вид цены" перейдите на закладку "Пороги срабатывания".
  • В табличной части добавлена строка "Все ценовые группы".
  • В графе "Порог срабатывания..." укажите процент порога срабатывания для всех ценовых групп.

Как настроить разные пороги срабатывания для различных ценовых групп?
  • В табличной части "Пороги срабатывания" нажмите пиктограмму "Добавить" или нажмите кнопку INS.
  • В появившейся новой строке выберите из списка "Ценовые группы" ту ценовую группу, для которой порог срабатывания отличается от порога срабатывания, который установлен для всех ценовых групп.
  • Введите порог срабатывания для конкретной ценовой группы.
