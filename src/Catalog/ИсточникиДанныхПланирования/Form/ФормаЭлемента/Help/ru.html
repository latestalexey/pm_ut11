Как заполняется информация в новом источнике данных?

Для нового источника данных для планирования необходимо задать его наименование и те правила, по которым будут формироваться данные для источника данных. Правила задаются с помощью схемы компоновки данных (кнопка "Настройка"). После настройки правил формирования данных для источника данных в поле "Описание" укажите полное описание тех правил, по которому формировались данные для источника, в том числе и список тех предустановленных отборов, которые были установлены для источника данных планирования.

В соответствии с типом источника должны быть определены те параметры указания периода, которые будут доступны при настройке данного источника при заполнении плана продаж и внутреннего потребления.

  • За период. При заполнении плана для источника указывается тот период, за который будут выбираться данные источника. Например, объем продаж товаров за прошлый месяц, или количество планируемого поступления товаров в течении следующего месяца.
  • На дату. При заполнении плана для источника указывается дата, на которую формируются данные источника. Например, остатки товаров на начало месяца планирования.
  • Без ограничения. При заполнении плана для источника не предусмотрено никаких ограничений по датам, будут заполнены все хранящиеся в информационной базе данные. Такую возможность следует использовать для тех источников данных, которые не имеют периодичности. Например, информация о минимальном остатке товаров из регистра "Товарные ограничения".

Как открыть форму настройки для определения правила заполнения данных для источника планирования?

В форме источника нажмите на кнопку "Настройка". Будет открыта форма для настройки правил заполнения данных в источнике.

При настройке правил формирования источника данных используются система компоновки данных. Пользователю предоставляется упрощенная схема настройки. С помощью упрощенной схемы настройки пользователь может редактировать следующие настройки правил формирования источника данных.

  • Задавать предопределенные параметры отбора данных в соответствии с параметрами, которые определены схемой компоновки источника данных и задавать те параметры отбора, которые будут доступны для быстрого отбора пользователю при использовании источника данных при заполнении плана-прогноза ("Отбор").
  • Задавать значение параметров, если это предусмотрено схемой компоновки источника данных (Параметры).
  • Производить сортировку данных по выбранным параметрам (Порядок).

Важно. При формировании нового источника данных схема компоновки данных не загружается. В соответствии с назначением источника данных пользователь должен выбрать нужную схему компоновки данных из списка поставляемых шаблонов ("Выбрать шаблон") или создать свою схему компоновки данных, используя систему компоновки данных ("Редактировать схему компоновки данных").

Как выбрать шаблон для настройки правил заполнения данных в источнике планирования?

  • В форме настройки источника данных нажмите на кнопку "Выбрать шаблон" .
  • Выберите из выпадающего списка нужный шаблон.
  • Нажмите на кнопку "ОК".
  • Будет загружен шаблон схемы компоновки данных. В соответствии с выбранным шаблоном будет заполнена информация в списке доступных полей для отбора.

В программе предусмотрены следующие варианты шаблонов для настройки источников данных.

  • Продажи. Данная схема компоновки данных выбирается в тех случаях, если в качестве источника планирования используются данные о продажах товаров за определенные периоды.
  • Свободные остатки. Данная схема компоновки данных выбирается в тех случаях, если в планах необходимо учитывать информацию об остатках товаров. В этом случае в список источников данных следует добавить источник данных, который будет учитывать свободные остатки товаров на определенную дату.
  • Заказы клиентов.Данная схема компоновки данных выбирается в тех случаях, если в планах необходимо учитывать информацию о тех заказах, по которым товар еще не был отгружен клиенту. Данные для источника такого типа должны заполняться за определенный период. Например, при составлении плана можно учитывать все планируемые отгрузки товаров клиентам в следующем месяце.
  • Заказы поставщикам. Данная схема компоновки данных выбирается в тех случаях, если в планах необходимо учитывать информацию о товарах, по которым ожидается поставка товаров от поставщика. Данные для этого источника должны заполняться за определенный период. Например, при составлении плана можно учитывать все планируемые поставки товаров от поставщиков в следующем месяце.
  • Заказы на внутреннее потребление. Данная схема компоновки данных выбирается в тех случаях, если в планах необходимо учитывать информацию о товарах, которые необходимо отгрузить на внутренние нужды подразделений. Данные для источника такого типа должны заполняться за определенный период.
  • Заказы на перемещение (отгрузка). Данная схема компоновки данных выбирается в тех случаях, если в планах необходимо учитывать информацию о товарах, которые необходимо отгрузить на другие склады предприятия. Данные для этого источника также должны заполняться за определенный период.
  • Заказы на перемещение (поступление). Данная схема компоновки данных выбирается в тех случаях, если в планах необходимо учитывать информацию о товарах, поступление которых планируется на другие склады предприятия. Данные для такого типа источника также должны заполняться за определенный период.
  • Заказы на сборку (отгрузка). Данная схема компоновки данных выбирается в тех случаях, если в планах необходимо учитывать информацию о товарах ( комплектующих, комплектах), которые необходимо отгрузить в процессе сборки (разборки). Данные для такого типа источника должны заполняться за определенный период.
  • Заказы на сборку (поступление). Данная схема компоновки данных выбирается в тех случаях, если в планах необходимо учитывать информацию о товарах (комплектах, комплектующих), которые поступят на склад предприятия после выполнения процесса сборки (разборки) . Данные для такого типа источника должны заполняться за определенный период.

Как указать при настройке источников данных тот список полей, по которым пользователь сможет производить отбор данных в источнике?

Список полей, по которым пользователь может производить отбор данных определяется в настройках источников данных на странице "Отборы". При загрузке шаблона схемы компоновки данных список полей, по которому пользователь может произвести отбор данных, загружается автоматически в соответствии с выбранным шаблоном. Все поля, которые загружаются автоматически при выборе шаблона доступны для редактирования пользователем.

Пользователь, формирующий источники данных, может расширить список этих полей, добавив в список полей отбора поля из списка "Доступные поля".

Для добавления дополнительных полей в список выполните следующие действия.

  • Установите курсор на нужное дополнительное поле в списке.
  • Нажмите на кнопку "Выбрать" вверху списка.
  • Дополнительное поле будет добавлено в список полей отбора.

Для того, чтобы это поле было доступно для осуществления отбора данных по данному источнику при планировании выполните следующие действия.

  • Установите курсор на поле отбора в списке.
  • Нажмите на кнопку "Все действия" и выберите пункт "Свойства элемента пользовательских настроек".
  • В появившейся форме установите флаг "Включать в пользовательские настройки".
  • В поле "Представление" можно изменить название отбора, которое будет отображаться у пользователя. Если это поле не заполнено, то у пользователя будет отображаться то наименование, которое указано в названии отбора, представленном в списке.

Можно ли при настройке источника данных ограничить область его применения?

При настройке источника данных может возникнуть необходимость создать раздельные источники, например, по разным группам номенклатуры. Такая возможность может быть востребована в том случае, если планирование продаж различных групп товаров осуществляется различными менеджерами. При этом каждый менеджер должен отвечать только за планирование своей группы товаров.

Пример. Необходимо сформировать два источника данных для планирования товаров групп "Бытовая техника" и "Мебель". При использовании данных источников менеджеры не должны иметь возможность изменять группу товаров, по которым производится планирование.

В этом случае в параметрах отбора данных источника необходимо установить отбор по конкретной группе номенклатуры (Номенклатура -В группе- Бытовая техника) . Для того, чтобы менеджер не смог при планировании изменить отобр данных в источнике, следует исключить этот отбор из пользовательских настроек. Для исключения этого отбора из пользовательских настроек необходимо в свойствах пользовательских настроек убрать флаг "Включать в пользовательские настройки".

Как отредактировать схему компоновки данных?

Важно. Схема компоновки данных может быть отредактирована только в режиме "толстого клиента".
  • Для редактирования схемы компоновки данных нажмите на кнопку "Настройка" в форме источника.
  • В диалоговом окне упрощенных настроек нажмите на кнопку "Редактировать схему компоновки данных".

Пользователь может произвольным образом отредактировать текст и параметры запросов, используя правила работы со схемой компоновки данных.

Как сохранить правила формирования источника данных?
  • После настройки схемы компоновки данных (с использованием шаблонов или используя произвольные запросы) в диалоговой форме настройки нажмите на кнопку "Завершить редактирование".
  • В форме источника данных нажмите на кнопку "Записать и закрыть".

Будет сохранена информация об источнике данных с введенными правилами его формирования. В дальнейшем эта информация используется при составлении плана продаж и внутреннего потребления.

Как заполнить информацию в плане продаж и внутреннего потребления автоматически, используя те данные, которые хранятся в информационной базе (объем продаж, заказы клиентов, остатки и т.д.)?

Важно. Для автоматического заполнения информации в плане продаж и внутреннего потребления предварительно должны быть созданы источники данных, которые будут являться основой для заполнения данных в табличной части документа.
  • Для автоматического заполнения табличной части документа нажмите на кнопку "Заполнить".
  • В появившейся форме "Заполнение плана " необходимо определить правила заполнения данных (обновление или добавление данных), список источников, по которым будут формироваться данные и правила их совместного применения .
  • Полученные результаты могут быть изменены и округлены в соответствии с указанными правилами округления.
  • Состав сформированных данных можно ограничить, установив на них различные параметры отбора.
  • Полученное количество товаров можно автоматически распределить по датам отгрузки в соответствии с рабочими днями склада.
  • Для переноса данных в табличную часть документа нажмите на кнопку "Заполнить документ".
