Справочник "Сделки с клиентами"

Справочник "Сделки с клиентом" предназначен для регистрации и хранения информации о сделках, оформленных с клиентами. С помощью сделки можно регламентировать процесс продажи, то есть составить те правила (этапы продаж), с помощью которых можно проводить регламентацию процессов продаж. В то же время сделку можно использовать и в том случае, если процессы продаж на предприятии не регламентированы. В этом случае сделка служит хранилищем всей информации, которая происходит в рамках сделки : документов, взаимодействий, внешних присоединенных файлов и т.д.

В программе можно создавать различные виды сделок. Для каждого вида сделки определяется тип сделки, который определяет способ управления сделкой. Сделки могут быть трех типов.

  • Прочие непроцессные сделки – по сделкам такого типа можно будет фиксировать лишь факт начала и окончания работы по сделке, этапы по данным сделкам не указываются.
  • Сделки с ручным переходом по этапам продаж. Для таких сделок нужно вручную определить перечень этапов из общего классификатора "Этапы процессов продаж". Можно использовать уже существующие в классификаторе предопределенные элементы или добавить свои. Расширение набора этапов имеет смысл для получения более детальной воронки продаж. Настройка последовательности этапов выполняется в самом классификаторе элементами управления и одинакова для всех видов сделок. Переход от одного этапа к другому пользователь будет выполнять вручную, при этом возможен пропуск некоторых этапов, возврат на предыдущие этапы. В самом списке этапы не могут повторяться, это всего лишь допустимые этапы.
  • Сделки, управление которыми описано в системе бизнес-процессом. В текущей версии конфигурации для примера приведен вариант управления бизнес-процессом "Типовая продажа". Основное отличие такого способа управления от предыдущего в том, что последовательность перехода от одного этапа процесса управления к другому жестко регламентирована и описана картой маршрута бизнес-процесса. Пропустить какой-то этап нельзя. На каждом этапе соответствующим пользователям выдаются задачи и контролируется их выполнение.

В программе предусмотрена возможность планирования сделок. Планирование сделок подразумевает составление, выполнение и контроль плана действий по сделке. Такой вариант управления применяется для долгосрочных сделок с уникальным набором действий и уникальным результатом. В системе для каждой сделки можно составить план проекта, где расписать весь необходимый перечень задач проекта.

При оформлении новой сделки с клиентом можно зарегистрировать первичный интерес клиента, указать канал рекламного воздействия, в результате которого клиент обратился с намерением купить товар. В процессе работы для сделки устанавливается статус "В работе". В документах, которые оформляются в рамках сделки ("Коммерческое предложение", "Заказ клиента", "Реализация товаров и услуг") указывается информация о сделке. В дальнейшем можно проконтролировать процесс выполнения сделки и список всех документов, оформленных по сделке. В рамках сделки можно хранить информацию о всех взаимодействиях с клиентами, а также список внешних присоединенных файлов.

После того как все работы по сделке будут закончены, сделка должна быть закрыта. Явный признак закрытия сделки устанавливается для того, чтобы в любой момент времени можно было получить список сделок, который еще находится в работе, проанализировать его, выполнить корректирующие воздействия. Закрыта сделка может быть с одним из результатов: сделка выиграна, сделка проиграна. Если сделка проиграна, то записывается причина проигрыша. В дальнейшем статистика по проигранным сделкам анализируется с помощью отчета "Анализ причин проигрыша сделок".

Анализ эффективности сделок можно посмотреть в следующих отчетах.

  • Воронка продаж
  • Эффективность процессов продаж
  • Эффективность сделок с клиентом

Предусмотрена также возможность сравнительного анализа прибыли, полученной в результате оформления сделок с клиентом ("Выручка и себестоимость продаж") и анализа доходов и расходов по каждой сделке ("Анализ доходов и расходов").
